# Story 4.1: Workflow Export Compilers

Status: done

## Story

As a workflow user,
I want to export my workflow as Prompt / TypeScript / Python / Mermaid / JSON,
so that I can use workflows as documentation, share them as code, or visualize them as diagrams.

## Acceptance Criteria

1. **Given** a workflow exists **When** I call `flowaibuilder.export({ workflow_id, format: "prompt" })` **Then** the workflow is returned as a structured natural-language prompt describing each node, its config, its connections, and overall data flow.

2. **Given** a workflow exists **When** I call `flowaibuilder.export({ workflow_id, format: "typescript" })` **Then** the workflow is returned as executable TypeScript source using the (notional) flowAIbuilder SDK shape — nodes declared in topological order, connections wired by id, exported as a `workflow` object.

3. **Given** a workflow exists **When** I call `flowaibuilder.export({ workflow_id, format: "python" })` **Then** the workflow is returned as executable Python source mirroring the TypeScript shape (function/dict-based DSL).

4. **Given** a workflow exists **When** I call `flowaibuilder.export({ workflow_id, format: "mermaid" })` **Then** a `flowchart LR` Mermaid diagram is returned with one labeled node per workflow node and one edge per connection (using sourceHandle/targetHandle when present).

5. **Given** a workflow exists **When** I call `flowaibuilder.export({ workflow_id, format: "json" })` **Then** the raw workflow JSON (same shape as `GET /api/workflows/:id`) is returned.

6. **Given** the export dialog UI shell from Story 1.8 exists **When** I open it and pick any non-JSON format **Then** the preview area renders the actual compiled output for that format (no more "Coming in Epic 4" placeholder), and both **Copy to Clipboard** and **Download** buttons act on that compiled content (Download triggers a file download with an appropriate filename and extension).

7. **Given** an unknown format string is passed to the MCP tool or REST endpoint **Then** a descriptive error is returned listing the valid formats.

## Dev Notes

### Scope boundary

This story is the **compiler layer + endpoint + MCP tool + UI wire-up**. It is NOT a full SDK. The TS/Python output references a notional `@flowaibuilder/sdk` / `flowaibuilder` package — that runtime SDK does not exist and is out of scope. The generated code is meant to be readable and reproducible, not directly executable today. Add the comment header so users understand.

Story 1.8 already shipped the dialog shell, the format buttons, and the JSON path. This story REPLACES the placeholder branch and changes Copy semantics (now copies the displayed content, not always JSON). The 4 placeholder strings must be gone.

### Why server-side compilers (not in the UI)

- The MCP tool needs them too — they must live in the server package.
- Keeps the UI free of compilation logic and lets us evolve compilers without redeploying the UI.
- Tests are easier (pure functions, no React).

### Compiler conventions

- All compilers are **pure functions** of `Workflow`. No DB access, no I/O, no async.
- Topological sort is best-effort (Kahn). Cycles → fall back to insertion order + warning comment. Never throw from a compiler — only `compileWorkflow` validates the format string.
- Mermaid node ids must be sanitized: `id.replace(/[^A-Za-z0-9_]/g, '_')`. Workflow node ids from `nanoid()` already match, but user-imported ids may not.
- When emitting TS object literals from JSON config, post-process unquoted keys with the regex above. Reserved JS words as keys are unlikely in node configs but acceptable to leave quoted.
- Python literal helper:
  - `null` → `None`
  - `true` → `True`, `false` → `False`
  - strings → JSON-encoded then swap surrounding `"` (Python accepts JSON string syntax)
  - dicts/lists → recurse with 4-space indent

### What NOT to do

- Do NOT build a real `@flowaibuilder/sdk` runtime package. The TS/Python output is a notional shape with a header comment explaining as much.
- Do NOT add validation that the compiled TS/Python is *executable*. It's meant to be readable + reproducible.
- Do NOT add new node types or modify the `Workflow` shape — compilers consume the existing shared types only.
- Do NOT call any AI/LLM for the prompt format — it's a deterministic, structured Markdown template.
- Do NOT add streaming, pagination, or progress UI to the dialog — workflows are small enough that a single request is fine.
- Do NOT add auth checks — Epic 5.
- Do NOT broadcast WebSocket events for export calls.
- Do NOT touch the MCP fix-handler registration block in `mcp/index.ts`.
